카테고리 **$1**, 난이도 **$2**에 맞는 퀴즈 문제를 새로 생성하여 `src/data/questions.ts`에 추가해줘.

## 사전 준비

1. `src/data/questions.ts`를 읽어 현재 최대 id를 확인한다.
2. `src/types/quiz.ts`를 읽어 Question 인터페이스를 확인한다.

## 입력 파라미터

- **$1 — 카테고리**: `popsong` | `worldmap` | `movie` | `general` 중 하나  
  (한글 입력도 허용: 팝송→popsong, 세계지도→worldmap, 외국영화→movie, 일반상식→general)
- **$2 — 난이도**: `easy` | `medium` | `hard`  
  (한글 입력도 허용: 쉬움→easy, 보통→medium, 어려움→hard)

파라미터가 비어 있거나 유효하지 않으면 작업을 중단하고 올바른 값을 안내한다.

## 문제 생성 규칙

난이도별 기준:
- **easy**: 대중적으로 잘 알려진 사실, 오답이 명확히 구분되는 선택지
- **medium**: 해당 분야 관심자라면 알 만한 사실, 유사한 선택지 포함 가능
- **hard**: 마니아 수준의 세부 지식, 헷갈리기 쉬운 선택지 구성

생성할 문제 수: **3문제**

각 문제는 아래 TypeScript 구조를 따른다:
```ts
{
  id: number;               // 현재 최대 id + 순번
  category: '$1';           // 입력받은 카테고리
  question: string;         // 질문 (조건·범위 명시 포함)
  options: [string, string, string, string]; // 선택지 4개
  answerIndex: 0 | 1 | 2 | 3;
  explanation: string;      // 정답 근거 + 출처·기준 명시 (필수)
}
```

## CLAUDE.md 검증 — 생성 전 각 문제에 대해 반드시 확인

생성한 문제를 파일에 추가하기 **전에** 아래 4가지를 스스로 점검한다.

1. **정답이 하나뿐인가?**  
   - 다른 해석이 가능한 선택지가 있으면 질문에 조건을 추가하거나 선택지를 교체한다.

2. **최상급 표현에 기준이 있는가?**  
   - '가장 큰', '최초의', '최장' 등의 표현은 측정 기준(면적·인구·흥행수익 등)을 질문 또는 explanation에 명시한다.

3. **시간과 범위가 명확한가?**  
   - 변할 수 있는 정보(인구·순위·기록 등)는 질문에 연도·기준 시점을 직접 명시한다.  
   - 지리적·분류적 범위도 필요 시 한정한다.

4. **교차 검증했는가?**  
   - 논란이 있는 내용은 explanation에 "주류 학설 기준", "전통적 기준" 등 출처 근거를 명시한다.  
   - 의심스러운 사실은 2개 이상의 근거로 확인한다.

검증에서 탈락한 문제는 수정 후 재검증한다. 수정으로도 해결되지 않으면 해당 문제를 폐기하고 다른 문제로 교체한다.

## 파일 수정

검증을 모두 통과한 문제만 `src/data/questions.ts`에 추가한다.

- 해당 카테고리의 마지막 문제 뒤에 삽입한다.
- 카테고리 주석 블록(`// ── 카테고리명 ──`)이 있으면 그 안에 삽입하고, 없으면 파일 끝(`];` 앞)에 추가한다.
- 기존 코드 스타일(들여쓰기, 따옴표, 후행 쉼표)을 그대로 유지한다.

