# PRD Step 2 — 레시피 생성

## 개요

Step 1에서 인식된 식재료 목록을 바탕으로 `qwen/qwen3.6-plus` 텍스트 모델을 호출하여 맞춤 레시피를 생성하고 사용자에게 표시한다.

---

## 목표

- Step 1 재료 목록을 기반으로 레시피 추천 API 호출
- 생성된 레시피를 구조화된 UI로 표시
- 레시피 저장(Step 3) 진입점 제공

---

## 기술 스택

| 항목 | 선택 |
|------|------|
| 프레임워크 | React + TypeScript |

## 기능 상세

### 1. 재료 목록 로드

- `sessionStorage`에서 `fridge_ingredients` 읽기
- 없으면 Step 1으로 리다이렉트

### 2. 생성 옵션

| 옵션 | 값 | 기본값 |
|------|----|--------|
| 인분 수 | 1 / 2 / 3 / 4 | 2 |
| 조리 시간 | 빠름(15분 이내) / 보통(30분) / 여유(1시간) | 보통 |
| 식이 제한 | 없음 / 채식 / 글루텐프리 | 없음 |
| 레시피 수 | 1~3개 | 1 |

### 3. API 호출 (Qwen 텍스트)

- 모델: `qwen/qwen3.6-plus`
- `max_tokens`: 1500
- 스트리밍: `stream: true` (실시간 타이핑 효과)

**시스템 프롬프트:**
```
너는 전문 요리사야. 주어진 재료로 만들 수 있는 실용적인 레시피를 추천해줘.
반드시 다음 JSON 형식으로만 응답해:
{
  "recipes": [
    {
      "name": "요리 이름",
      "time": "조리 시간",
      "servings": 인분수,
      "ingredients": ["재료1 분량", "재료2 분량"],
      "steps": ["1단계", "2단계"],
      "tip": "요리 팁 (선택)"
    }
  ]
}
```

**유저 프롬프트 템플릿:**
```
냉장고에 있는 재료: {ingredients}
인분 수: {servings}인분
조리 시간: {time}
식이 제한: {diet}
레시피 {count}개를 추천해줘.
```

### 4. 응답 표시

- 스트리밍 응답을 실시간으로 파싱하여 타이핑 애니메이션 효과 적용
- JSON 파싱 완료 후 구조화된 카드 UI로 전환
- 파싱 실패 시 원문 마크다운 렌더링으로 폴백

### 5. 다시 생성

- 같은 재료로 다른 레시피 재요청
- 이전 결과와 함께 표시 (최대 3세트 유지)

## 레시피 데이터 구조

```typescript
type Recipe = {
  id: string;            // crypto.randomUUID()
  name: string;          // 요리 이름
  time: string;          // 조리 시간
  servings: number;      // 인분 수
  ingredients: string[]; // 재료 목록
  steps: string[];       // 조리 순서
  tip?: string;          // 요리 팁
  sourceIngredients: string[]; // 인식에 사용된 원본 재료
  createdAt: string;     // ISO 8601
};
```

## 오류 처리

| 상황 | 처리 방법 |
|------|----------|
| sessionStorage 재료 없음 | Step 1으로 자동 이동 |
| API rate limit (429) | "잠시 후 다시 시도해 주세요" + 재시도 버튼 |
| JSON 파싱 실패 | 마크다운 원문 렌더링으로 폴백 |
| 네트워크 오류 | 오류 메시지 + 재시도 버튼 |
