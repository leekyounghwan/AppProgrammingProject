# PRD Step 1 — 냉장고 이미지 인식

## 개요

사용자가 냉장고 사진을 업로드하면 `google/gemma-3-27b-it:free` 비전 모델을 통해 식재료 목록을 자동으로 추출하는 기능을 구현한다.

---

## 목표

- 이미지 업로드 UI 제공
- OpenRouter API(Gemma 비전 모델)를 호출하여 식재료 인식
- 인식된 재료 목록을 다음 단계(Step 2)로 전달할 수 있는 상태로 저장

---

## 기능 상세

### 1. 이미지 업로드

- 지원 형식: `image/jpeg`, `image/png`, `image/webp`
- 최대 파일 크기: 10MB
- 업로드 방식: 파일 선택 다이얼로그 + 드래그 앤 드롭
- 업로드 후 미리보기 표시

### 2. API 호출 (Gemma 비전)

- 이미지를 Base64로 인코딩하여 `data:image/...;base64,...` 형식으로 전달
- 프롬프트:
  ```
  이 냉장고 사진에서 보이는 식재료를 모두 찾아줘.
  결과는 JSON 배열 형식으로만 반환해줘.
  예시: ["당근", "계란", "우유", "양파"]
  ```
- 모델: `google/gemma-3-27b-it:free`
- `max_tokens`: 300

### 3. 응답 파싱

- 응답 텍스트에서 JSON 배열 추출 (정규식 파싱)
- 파싱 실패 시 원문 텍스트를 줄 단위로 분리하여 재료 목록 구성
- 인식된 재료를 태그(chip) 형태로 표시

### 4. 재료 수정

- 인식 결과에서 재료 삭제 가능 (X 버튼)
- 재료 직접 추가 가능 (텍스트 입력 + Enter)

### 5. 상태 전달

- 최종 재료 목록을 `sessionStorage`에 저장 (`fridge_ingredients` 키)
- "레시피 추천 받기" 버튼 클릭 시 `/recipe` 페이지로 이동

## 오류 처리

| 상황 | 처리 방법 |
|------|----------|
| 파일 크기 초과 | 업로드 차단 + 안내 메시지 |
| 지원하지 않는 파일 형식 | 업로드 차단 + 안내 메시지 |
| API rate limit (429) | "잠시 후 다시 시도해 주세요" 안내 |
| API 오류 (기타) | 오류 메시지 표시 + 재시도 버튼 |
| 재료 인식 결과 없음 | "재료를 인식하지 못했습니다. 직접 입력해 주세요" 안내 |
